       • Direct position hold via PID
       └─ Update motor PWM
    
    Accumulate error for integral term
    Log telemetry to CSV
    Sleep 5ms → repeat

Cubic spline trajectory generation

The math behind smooth acceleration: solving a 4×4 linear system to create continuous paths between waypoints.

The Problem

Linear interpolation between waypoints creates jerky motion: acceleration jumps at each waypoint. Motors overshoot due to discontinuous commands.

The Solution

3rd-order polynomial cubic spline: ensures zero velocity at waypoints and continuous acceleration. Real smooth paths. 

// Spline coefficient computation (GimmeCoeff)
Boundary conditions: Start at pos₀, End at pos_f, zero velocity at both
Cubic equation: p(t) = c₀ + c₁·t + c₂·t² + c₃·t³

System of equations (4×4 linear):
  p(0) = pos₀           →  c₀ = pos₀
  ṗ(0) = 0              →  c₁ = 0
  p(T) = pos_f          →  c₀ + c₁·T + c₂·T² + c₃·T³ = pos_f
  ṗ(T) = 0              →  c₁ + 2·c₂·T + 3·c₃·T² = 0

Solve via numpy.linalg.solve(A, b) for c₀, c₁, c₂, c₃

// Real-time trajectory evaluation
Every 5ms: desired_pos = TargetDist(elapsed_time, coeff)

Result: 24 waypoint transitions, each with continuous acceleration. System response is predictable and tunable.

Engineering highlights: code & control design

Technical decisions that make this more than just a working demo.

Cubic spline trajectory generation

Problem: Linear interpolation between waypoints creates discontinuous accelerations, causing jerky motion and motor overshoot.

Solution: 3rd-order cubic spline with zero-velocity boundary conditions. Solves a 4x4 linear system at each waypoint transition to guarantee smooth, continuous-acceleration paths.

Code: `GimmeCoeff()` computes spline coefficients; `TargetDist()` evaluates the cubic polynomial in real-time for each motor independently.

Cascaded PI/PID control loops

Problem: 5 motors with different dynamics (3 linear, 2 rotational). Single gain set won't work across all axes.

Solution: Tuned individual Kp/Ki/Kd for each motor. Joints 1,2,3 use PI; Joint 4,5 use PID. Error accumulation (esum) drives integral term; position feedback from encoders.

Implementation: 5ms loop cycle. Read encoder → compute error → accumulate error → output PWM. Closed-loop ensures tracking fidelity.

Waypoint sequencing & task orchestration

Challenge: 24-waypoint harvesting sequence requires safe, repeatable motion without hardcoded delays or manual intervention.

Design: Waypoint list + transition times (tf array). Loop over waypoints, generate splines on-the-fly, run closed-loop control until target reached.

Result: Deterministic, testable sequence. Easy to modify waypoints/times without recompiling control loop. Soft e-stop via signal handler for safety.
